# IG Social CRM - V1.0.0
**Release Date:** March 1, 2026

## Included in V1
- Supabase-backed auth with email/password login and signup:
  - `supabase.auth.signInWithPassword({ email, password })`
  - `supabase.auth.signUp({ email, password })`
- Client-side auth form validation (valid email + minimum password length).
- Protected app area (`/app`) enforced by middleware session checks.
- Dashboard view with lead summary counts.
- Kanban pipeline with:
  - Drag-and-drop stage updates.
  - Lead detail panel editing.
  - Debounced autosave status feedback (`Saving...`, `Saved`, `Error`).
- CSV import flow:
  - UI at `/app/import`
  - API endpoint at `/api/import-leads`
  - Identity-aware upsert keyed on `(agent_id, ig_username)` after handle resolution from `ig_username/email/phone/external_id`.
- Public questionnaire intake flow:
  - UI at `/intake`
  - API endpoint at `/api/intake`
  - Upsert strategy keyed on `(agent_id, ig_username)` with synthetic identity fallback.
- Logout flow and auth redirect back to `/auth`.
- Debug routes for session and RLS checks (`/_debug/supabase`, `/ping`).

## Excluded from V1
- Admin panel and role management.
- Billing/subscription features.
- Native analytics/reporting dashboards beyond basic lead counts.
- Public API surface for third-party clients.
- In-repo automated CI pipeline and comprehensive test suite.

## Architecture
- Frontend/backend framework: Next.js (App Router).
- Hosting/runtime target: Vercel.
- Backend services: Supabase Auth + Postgres via Supabase clients.
- Client usage:
  - Browser auth/data operations use `createBrowserClient`.
  - Middleware/server operations use `createServerClient`.
- Required runtime environment variables:
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

## Database and RLS Expectations
- Core table used by the app: `leads`.
- Key fields used by UI/import flows:
  - `id`, `agent_id`, `ig_username`, `intent`, `timeline`, `lead_temp`, `source`, `notes`, `stage`, `time_last_updated`.
- Data isolation expectation for production:
  - RLS must restrict read/write access to owned records using `agent_id = auth.uid()`.
- Import upserts rely on a unique conflict target:
  - `onConflict: "agent_id,ig_username"`.

## CSV Import Behavior
- Accepts multipart upload with field name `file`.
- Requires at least one identity column per row:
  - `ig_username` or `email` or `phone` or `external_id`.
- Common supported headers:
  - `ig_username`, `email`, `phone`, `external_id`, `first_name`, `last_name`, `full_name`, `tags`, `intent`, `timeline`, `lead_temp`, `source`, `notes`, `stage`.
- Header normalization:
  - Lowercased and spaces converted to underscores.
- Row normalization and validation:
  - Handles/emails/phones are normalized.
  - Rows missing all identity fields are rejected as row errors.
  - Duplicate resolved identities in the same file are skipped.
- Upsert behavior:
  - Writes per authenticated user with `agent_id = user.id`.
  - Defaults missing `stage` to `New`.
  - Updates `time_last_updated` on write.
- API response summary:
  - `inserted`, `updated`, `skipped`, `errors[]`.
